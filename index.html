Hackathon project demos illustrate the creative thinking and team work applied to building unique applications from the IM&G Common Application Framework (CAF) components.

Best App - Solves non-trivial and relevant business need

Summary: Our app, named “CAF file system”, is a remote virtual file system based on CAF Storage Service.

CAF Components Used: CAF Storage Service

Other Components Used: FUSE (File System in user space)

Data Source: none

Description: This app is designed to implement a "File System" style APIs on top of existing CAF Storage APIs. For Admin user, it can provide a better analysis and visualization in CAF Storage Service. Also, for Customer user, it gives a simple and convenience way to utilize CAF Storage Service. As the expectation for "CAF file system", we plan to provide:

  • A series of Dashboards with various KPIs visualization to CAF Storage Service
  • Integrate "CAF file system" into Linux with FUSE.
  • Command Line Interface, which is similar with "Hadoop File System CLI"
  • "File System" Java APIs based on CAF Storage APIs

Most innovative use of CAF APIs

Description: Objective of the project:

  • Gather digital media sources about an event or an organization or a brand.
  • Store the media on cloud using CAF storage.
  • Process the media and develop a usable abstract to analyze the big data, use entity extraction, stemming, etc.
  • Search big datasets on social media, blogs, news feeds, search engines, etc. using the abstract.
  • Develop useful insights about that specific event or brand. Come up with statistical/data models.
  • Render the models in graphs visually for ease of use
  • Make the complex data simpler!

Honorable Mention

Description: Our project is called Cloud Storage Multiplier(CSM). This is an adapter application that works for cloud storage services such as Dropbox, Google Drive and OneDrive. It utilizes CAF Storage Service’s block-level deduplication feature to help the users significantly increase their data transmission speed and largely save the online space. Small business companies will benefit from this application most because we can help to reduce a large amount of duplicate data across the whole company. At the same time, CSM exponentially increases the company’s file sharing capability because the shared file stub is only ~KB instead of ~MB or even ~GB. In one word, the combined space of duplicate data and shared files, and the time of transmitting duplicate data are what CSM can save for the users.

CAF Components Used: CAF Storage and CAF IDM services

Other Components Used: N/A

Data Sources Used: Dropbox
