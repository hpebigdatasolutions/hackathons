Southborough March 2016 Demos

The Southborough Hackathon took place in March 2016. The teams were tasked with building unique applications from the IM&G Common Application Framework (CAF) components.

Besides the CAF components, Hackathon teams took advantage of IDOL, HPE Elements, web data, and a host of other resources available to build applications within a 24 hour window.

Recognized Projects

Best App - Solves non-trivial and relevant business need

Summary: Our app, named “CAF file system”, is a remote virtual file system based on CAF Storage Service.

CAF Components Used: CAF Storage Service

Other Components Used: FUSE (File System in user space)

Data Source: none

Description: This app is designed to implement a "File System" style APIs on top of existing CAF Storage APIs. For an Admin user, it provides an improved analysis and visualization of the data stored by CAF Storage Service. Also, for a Customer user, it provides a simple and convenient way to utilize the CAF Storage Service. The overall goals of the project were:

  • A series of Dashboards with various KPIs visualizations of CAF Storage Service data
  • Integration of "CAF file system" into Linux with FUSE.
  • Command Line Interface, similar to Hadoop File System CLI
  • "File System" Java APIs based on CAF Storage APIs

Best GUI - Completeness, clarity, simplicity, aesthetic value

Summary: The goal of this project was to combine a search engine with the CAF Policy engine in order to create a powerful and highly configurable search. The application is powerful because it can work with billions of search results. It is highly configurable because it ties into the CAF Policy engine and can also be used with different search providers like Google, IDOL, or Bing. For the demo, HPE Elements is used to display the search results and create graphs that describe what policies are matched by the query results.

CAF Components Used: CAF Policy Engine, HPE Elements

Other Components Used: Tomcat, Java

Data Sources Used: Google search

Most innovative use of CAF APIs

Description: Objective of the project:

  • Gather digital media sources about an event or an organization or a brand.
  • Store the media on cloud using CAF storage.
  • Process the media and develop a usable abstract to analyze the big data, use entity extraction, stemming, etc.
  • Search big datasets on social media, blogs, news feeds, search engines, etc. using the abstract.
  • Develop useful insights about that specific event or brand. Come up with statistical/data models.
  • Render the models in graphs visually for ease of use
  • Make the complex data simpler!

CAF Components Used: Storage, IDM, Policy Workflow, Speech-to-text, Elements

Other Components Used: HAVEn on Demand, Google Trends API, New York Times feed, Twitter APIs

Data Sources Used: Twitter feed, Event press releases, Presidential debate transcripts

Most interesting / fun app

Summary: Our goal was to write software that could leverage CAF services to analyze a video for usable words, pick those words out, and rearrange them into new audio clips. Originally, we wanted these videos to be analyzed dynamically, and include a variety of different voices. Early on in development we realized this was impractical, so we reduced scope to a single voice (Hillary Clinton), with the videos analyzed ahead of time, but still dynamically re-combined from the prepared words. We expected the results to be hilarious, and they were.

CAF Components Used: CAF Storage, CAF Policy, Speech Worker

Other Components Used: Java Audio Library

Data Sources Used: Hillary Clinton campaign speech videos, found here

Honorable Mention

Description: Our project is called Cloud Storage Multiplier(CSM). This is an adapter application that works for cloud storage services such as Dropbox, Google Drive and OneDrive. It utilizes CAF Storage Service’s block-level deduplication feature to help the users significantly increase their data transmission speed and largely save the online space. Small business companies will benefit from this application most because we can help to reduce a large amount of duplicate data across the whole company. At the same time, CSM exponentially increases the company’s file sharing capability because the shared file stub is only ~KB instead of ~MB or even ~GB. In summary, the combined space of duplicate data and shared files, and the time of transmitting duplicate data are what CSM can save for the users.

CAF Components Used: CAF Storage and CAF IDM services

Other Components Used: N/A

Data Sources Used: Dropbox
